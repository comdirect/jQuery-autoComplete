autoCompleteCD

An extremely lightweight completion suggester plugin for jQuery.

This is a fork of https://github.com/Pixabay/jQuery-autoComplete

Download   View on GitHub

Overview and Features

Released under the MIT License. Source on Github (changelog). Compatible with jQuery 1.7.0+ in Firefox, Safari, Chrome, Opera, Internet Explorer 7+. No dependencies except the jQuery library.

  • Lightweight
  • Fully flexible data source
  • Delay and minimum character settings
  • Callbacks
  • Note: The cache functionality isn't available in this fork. Maybe it will be supplemented some time.

This plugin was originally developed by and for Pixabay.com - an international repository for free Public Domain images. They have implemented this piece of software in production and share it - in the spirit of Pixabay - freely with others.

This fork is to allow a) muliple choices, and b) additional content which is not in the list of choices. In the result, you can comma-separate any mixture of choices and text in the auto-complete text field. In the example above, you can put stuff like Java, Brainfuck, Javascript, where Java and Javascript are choices, but Brainfuck not.

Usage

Include the stylesheet jquery.auto-completeCD.css in the <head> section of your page - and the JavaScript file jquery.auto-completeCD.js after loading the jQuery library. autoComplete accepts settings from an object of key/value pairs, and can be assigned to any text input field. 

$(selector).autoCompleteCD({key1: value1, key2: value2});

Settings

PropertyDefaultDescription
choiceObjects[]The array with choice-objects.
In the most simple case, this is an array of strings like ['ch1', 'ch2', 'ch3']. In this case, you can use the default-function for source()
source(term, response)function Callback function to pick choices from choiceObjects as suggestions to autoCompleteCD. The callback gets three arguments:
  • searchTerm, which refers to the value currently in the text input.
  • choiceObjects, as given in the choicePbjects.
  • A suggestFn callback, which expects a single argument: the data to suggest to the user. This data must be an array of filtered suggestions based on the provided term:
    ['suggestion 1', 'suggestion 2', 'suggestion 3', ...]
minChars2Minimum number of characters a user must type before a search is performed.
delay150The delay in milliseconds between when a keystroke occurs and when a search is performed. A zero-delay is more responsive, but can produce a lot of load.
cachefalseNot supported, always off.
menuClass'' Custom class/es that get/s added to the dropdown menu container.
Example: { menuClass: 'class1 class2' }
renderChoiceElementfunction

A function that gives you control over how suggestions are displayed. Default:

function (choiceObject, searchTerm){
    searchTerm = searchTerm.replace(/[-\/\\^$*+?.()|[\]{}]/g, '\\$&');
    var re = new RegExp("(" + searchTerm.split(' ').join('|') + ")", "gi");
    return '<div class="autocompleteCD-suggestion" data-val="' + choiceObject + '">'
        + choiceObject.replace(re, "<b>$1</b>") + '</div>';
}

Demos

Searching in local data

This plugin was designed mainly with AJAX requests in mind, but it may be used with local data, as well. Connecting the autocompletion suggester to an array of strings can be done in the source function like so: 

$('input[name="q"]').autoCompleteCD({
    minChars: 2,
    choiceObjects: ['ActionScript', 'AppleScript', 'Asp', ...]
});

The source function iterates through an array of (local) choices and we return a new array containing all (lowercased) matches. Simple as that.

AJAX requests DEMO NOT ADOPTED TO FORK!

AJAX requests may come with very different requirements: JSON, JSONP, GET, POST, additionaly params, authentications, etc. In order to keep the source code small while retaining full flexibility, we decided to only use a simple callback function as the source for suggestions. Make your AJAX call inside this function and return matching suggestions with the response callback: 

$('input[name="q"]').autoCompleteCD({
    source: function(term, response){
        $.getJSON('/some/ajax/url/', { q: term }, function(data){ response(data); });
    }
});

The AJAX call in this example needs to return an array of strings. The callback response must always be called, even if no suggestions are returned or if an error occured. This ensures the correct state of the autoCompleteCD instance.

Optimizing AJAX requests DEMO NOT ADOPTED TO FORK!

All search results are cached by default and unnecessary requests are prevented in order to keep server load as low as possible. To further reduce server impact, it's possible to abort unfinished AJAX requests before starting new ones: 

var xhr;
$('input[name="q"]').autoCompleteCD({
    source: function(term, response){
        try { xhr.abort(); } catch(e){}
        xhr = $.getJSON('/some/ajax/url/', { q: term }, function(data){ response(data); });
    }
});

By typing along, the user may trigger one AJAX request after the other. With this little trick, we make sure that only the most current one actually gets executed - if not done so already.
